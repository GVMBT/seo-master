## Что в пакете

Три документа для **подготовки к Сессии 4B**. Оформлены как независимые файлы, чтобы команда A могла применять по отдельности в разное время.

### 1. `ARTICLE_FORMATS_LIBRARY.md`

Библиотека из **7 форматов статей**. Системный промпт вашего AI выбирает формат по типу темы и генерирует по соответствующему шаблону. Каждый формат использует **разные типы блоков API** — это ломает AI-шаблонность, которую индексаторы Яндекс/Google детектируют.

**Ключевая проблема, которую решает:** наши 3 sandbox-статьи имеют одинаковую структуру. В production-поток 3-5 статей/неделя такая шаблонность = SEO-катастрофа.

**Что применить в 4B:** добавить библиотеку в системный промпт AI + правила выбора формата.

**Приоритет:** высокий. Без этого production-статьи будут все похожи друг на друга.

### 2. `ARTICLE_LENGTH_POLICY.md`

Политика **длины 1500-2200 слов** с обоснованием минимума и максимума.

**Ключевая проблема, которую решает:** наши sandbox-статьи были 630-720 слов. Это «thin content» по критериям SEO — Яндекс и Google депрессируют.

**Что применить в 4B:** добавить в промпт жёсткие границы 1500-2200 + инструкцию по самопроверке длины.

**Приоритет:** высокий. Без этого SEO не начнёт работать.

### 3. `SESSION_JOURNAL.md`

Финальный журнал интеграции. Не требует действий команды A. Для истории и синхронизации состояния.

Включает:
- Полный список работающих endpoints (14 + logout)
- История всех hotfixes (1.1.1 → 1.1.4)
- Все 3 итерации 4A с метриками
- Публикация первой production-статьи
- Критерии для перехода с sandbox-first на production-default

## Короткая формулировка для промпта команды A

Если вы хотите **максимально короткое прошивание** длины и форматов в ваш системный промпт (не читать целиком 2 моих документа):

```
LENGTH: 1500-2200 слов в блоках p+h2+h3+list+quote+callout.
        НЕ добивай водой. При недостатке слов — расширь примеры из KB.

FORMATS (выбирай по теме):
1. Гайд-инструкция (как выбрать/установить) → ol + table + callout
2. Сравнение (vs/или/что лучше) → table + 2 product + callout
3. Use-case (ванная/фасад/спальня) → 2+ product + h3 Q&A секция
4. Обзор коллекции (N декоров) → 5+ product с h3-подводками
5. Отраслевой (для отелей/офисов) → table эконом + ul требования
6. Q&A (10 вопросов о X) → 8+ h3 вопросов, минимум h2
7. Тренды/стили (скандинавский/минимализм) → 3+ product + callout

Если тема не подходит ни под один — Формат 3.
СТРОГО использовать обязательные блоки каждого формата.
H2-заголовки не повторять между статьями.
НЕ начинать с "В этой статье разберём".
```

Полный текст в `ARTICLE_FORMATS_LIBRARY.md` и `ARTICLE_LENGTH_POLICY.md`.
