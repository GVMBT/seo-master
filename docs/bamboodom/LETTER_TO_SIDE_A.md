Спасибо за быстрый отчёт по интеграции. **Поправили один важный косяк** до того, как вы успеете запустить smoke-тесты:

В первой версии endpoint'а описания (`description`) брались **только из `BLOG_META_FILE`** (overrides от админа). У нас сейчас этот файл практически пустой — кроме demo-записи у TK029P, которую вы заметили.

В обновлённой версии (см. ниже) логика такая:

```php

```

То есть приоритет: **админский override → описание из самого индекса**. И мы обогатили `article_index.json` описаниями **всех 1021 товара** (вытянули из schema.org разметки, статичных страниц `/texture/*.html` и т.д.).

## Качество описаний — что AI получит

Распределение по длине описания в новом `article_index.json`:

- **lighting**: 100-300 символов в среднем — конкретные тех.характеристики из schema.org производителя ERDU. Высокое качество.
- **wpc**: 150-250 символов — описания серий из дополнительной разметки на статичных страницах (`additionalProperty`). Высокое качество.
- **flex (старые 68/77)**: 80-100 символов — короткие meta description (типа «Гибкая керамика 3D Geometric Design. Толщина 5 мм, вес 7 кг/м²»). Среднее качество, но без галлюцинаций.
- **flex (новые 9/77)**: ~120 символов — шаблон по collection ("Гибкая керамика {name} — коллекция Rammed Earth, толщина 2-17 мм..."). Низкое качество, но безопасно.
- **reiki**: 150-200 символов — полные описания с упоминанием формы и применения. Высокое качество.
- **profiles**: 200-300 символов — детальные описания типов профилей с областью применения. Высокое качество.
