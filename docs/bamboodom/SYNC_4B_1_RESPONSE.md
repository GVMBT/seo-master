### 🟡 Вопрос по статье про спальню

Раз validator сказал что AI пишет 1267 слов вместо 1500 на третьей попытке retry — это **паттерн**. Не одна статья, не одна попытка. Означает что либо:

- **Промпт требует от AI добирать слова с ценой галлюцинаций**, которые ловит валидатор (TK042A, TK055C — AI попытался добрать количество артикулов, выдумывая новые)
- **Тема «WPC в спальне» бедна в knowledge base** — для неё AI просто не может найти 1500 слов правды

Предлагаю **в 4B.2** добавить в промпт правило:

```
Если на третьей попытке retry статья < 1500 слов — 
НЕ галлюцинировать факты, а:
1. Добавить секцию "Частые вопросы" (h3 + p, 4-6 пар) —
   использовать KB раздел 3 FAQ если он содержит релевантные вопросы
2. Детализировать технологию монтажа (раздел 1.1 / 1.2 KB)
3. Расширить секцию о применении с конкретными сценариями

Если после этого статья всё ещё < 1500 — 
вернуть ошибку "insufficient KB for topic", 
оператор переформулирует keyword.
```

Это **лучше чем галлюцинации**. Оператор B перевыберет тему, когда KB расширится — вернёмся к исходной.
