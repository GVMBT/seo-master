# Forbidden Semantic Corpus для Layer 3 валидатора

**Версия:** 1.0
**Дата:** 24.04.2026
**От:** сторона B (Claude, bamboodom.ru)
**Для:** сторона A, Сессия 4B, семантический валидатор (Claude Haiku)

---

## Назначение

Данный корпус — **список запрещённых семантических паттернов**, которые regex-валидатор (layer 1+2) не детектирует, но они нарушают:
- Knowledge base раздел 5 (forbidden claims) — запрещённые обещания
- Медицинское/санитарное законодательство — заявления про здоровье без сертификата
- Конкурентное право — сравнения с конкретными брендами
- Маркетинговые правила ФАС — необоснованные превосходные степени

AI-драфтер (Claude Sonnet 4.5) периодически **обходит буквальные запреты через перифразы**. Layer 3 ловит семантику, не буквы.

---

## Архитектурный принцип

Layer 3 **не заменяет** layer 1+2, а **дополняет**:
- **Layer 1 (regex)** — ловит буквальные «гипоаллергенно», «E1», «Г1», «Bostik», «Sika», «самый дешёвый», «скидка 70%»
- **Layer 2 (regex для чисел)** — ловит конкретные проценты и годы в контексте обещаний
- **Layer 3 (Claude Haiku)** — ловит перифразы, намёки, косвенные утверждения

Haiku получает текст блока + список **категорий** (ниже). Для каждой категории — решение **да/нет**: содержит ли блок такое нарушение? Если да — возвращает цитату и категорию.

## Промпт для Claude Haiku (layer 3)

```
Ты — валидатор статей для блога компании, продающей отделочные материалы.

Проверь следующий текст на наличие запрещённых семантических паттернов.
Для каждого блока верни JSON: {"violations": [{"category": "...", "quote": "...", "explanation": "..."}]}

Если нарушений нет — верни {"violations": []}.

```

## Категории запрещённых паттернов

### Категория 1: Медицинские/санитарные обещания

Косвенные утверждения о здоровье без сертификата.

**Буквальные формулировки (ловит layer 1):**
- «гипоаллергенно»
- «безопасно для детей»
- «антибактериальные свойства»

**Семантические перифразы (layer 3 ловит):**
- «подходит для людей, чувствительных к бытовой химии» ← итерация #1
- «безопасно для астматиков и аллергиков»
- «не вызывает раздражения дыхательных путей»
- «идеально для детской комнаты без риска для здоровья»
- «не накапливает патогенов»
- «природный материал, не влияющий на микроклимат»
- «материал дружелюбный к чувствительным людям»
- «подходит домам с животными / детьми» **в контексте безопасности**
- «подходит для астматиков»
- «используется в медицинских учреждениях» (если это не факт в KB)

**Как Haiku должен решать:** если текст **намекает** что материал улучшает здоровье или безопасен для группы (дети, аллергики, астматики, пожилые, беременные) — нарушение.

---

### Категория 2: Пожарная безопасность без сертификатов

Упоминание пожарных классов не имея их.

**Буквальные (layer 1 ловит):**
- «класс Г1»
- «КМ1»
- «не горит»
- «пожаробезопасный»

**Семантические перифразы:**
- «самозатухающий материал»
- «не поддерживает горение»
- «соответствует требованиям пожарной безопасности»
- «разрешён к использованию в общественных зданиях» (если нет сертификата)
- «материал с низкой горючестью»
- «устойчив к открытому пламени»
- «безопасен при высоких температурах»
- «можно монтировать рядом с камином» (если нет данных по температурам)
- «подходит для использования в саунах и банях» (где есть требования к огнестойкости)

**Как Haiku решает:** если текст **обещает** пожарные свойства, даже мягко — нарушение. Разрешено только: «материал обладает типичными для композитов характеристиками».

---

### Категория 3: Экологические сертификаты без основания

**Буквальные (layer 1 ловит):**
- «E1»
- «E0»
- «FSC»
- «LEED»
- «CE marking»

**Семантические перифразы:**
- «минимальная эмиссия формальдегида»
- «экологически чистый»
- «без вредных выделений»
- «не содержит фенолов» (если не проверено лабораторно)
- «сертифицирован для детских учреждений»
- «подходит для эко-интерьеров без ограничений»
- «соответствует европейским экологическим стандартам»
- «биоразлагаемый» (для композитов — обычно неправда)
- «изготовлен по технологиям, используемым в Европе»
- «zero-VOC»
- «без летучих органических соединений»

**Как Haiku решает:** экологичность должна быть **описательной** («содержит 80% натурального бамбука» — ok, это факт из KB). Утверждения о «соответствии стандартам» — нарушение.

---

### Категория 4: Конкретные бренды конкурентов

**Буквальные (layer 1 ловит при точных именах):**
- «Bostik»
- «Sika»
- «Ceresit»
- «Kleiberit»
- «Momentum»
- «КВГ»

**Семантические перифразы:**
- «клей от известных европейских производителей»
- «как в случае с популярными марками клея на строительных рынках»
- «универсальный клей, продающийся во всех магазинах»
- «немецкий клей известного производителя»
- «продукция Leroy Merlin / Петрович / OBI / Castorama»

**Как Haiku решает:** если текст упоминает конкретного производителя клея/инструмента/конкурентного материала — нарушение. Исключение: если это **сам Дизайн-Сервис** или если это **общеотраслевая терминология** (например «плиточный клей для наружных работ» — это тип, а не бренд).

### Категория 5: Необоснованные гарантии долговечности

**Буквальные:**
- «50 лет гарантии»
- «пожизненная гарантия»
- «гарантия качества»

**Семантические перифразы:**
- «прослужит десятилетиями»
- «не потеряет внешний вид за всю жизнь здания»
- «вечный материал»
- «не нужно обновлять никогда»
- «материал на века»
- «срок службы превышает 30 лет» (без обоснования из KB)
- «не требует замены на протяжении всей жизни»
- «надёжнее камня»
- «долговечнее натуральных материалов»

**Как Haiku решает:** конкретные сроки долговечности разрешены **только** если они есть в knowledge base. Общие фразы «долговечный», «надёжный» — **ok**. Численные обещания — **нарушение** если не из KB.

### Категория 6: Обещание цены / скидок в тексте

Критичное для Дизайн-Сервиса: **цены никогда не пишутся в текст статей** — они подставляются из API через product-блоки.

**Буквальные (layer 1):**
- «скидка 70%»
- «дешевле всех»
- «лучшая цена»
- «минимальная стоимость»

**Семантические перифразы:**
- «один из самых бюджетных вариантов на рынке»
- «доступно для большинства заказчиков»
- «по цене ниже средней»
- «значительно дешевле альтернатив»
- «экономный выбор»
- «недорогое решение»
- «специальная цена для постоянных клиентов»
- «акционное предложение»
- «оптимальное соотношение цены и качества» (клише, но в контексте продажи — нарушение)

**Как Haiku решает:** любое упоминание **цены в контексте оценки** (дешевле, выгоднее, экономно) — нарушение. Цены только через product-блок сервером.

### Категория 7: Обещание сроков доставки/монтажа

**Буквальные:**
- «доставка за 24 часа»
- «монтаж за 1 день»

**Семантические перифразы:**
- «быстрый монтаж под ключ»
- «материал доставляется в короткие сроки»
- «оперативная отгрузка»
- «монтаж не занимает много времени»
- «за пару дней ремонт закончен»
- «материал быстро появится у вас»
- «не придётся долго ждать»

**Как Haiku решает:** обещания сроков — **всегда** нарушение. Исключение: технические описания («клеевой монтаж быстрее обрешёточного» — ок как сравнение методов).

### Категория 8: Сравнения с отдельными категориями материалов

Утверждения, что WPC/flex **лучше** других материалов.

**Буквальные (layer 1):**
- «лучше ламината»
- «превосходит плитку»

**Семантические перифразы:**
- «в отличие от устаревшего МДФ»
- «современная альтернатива кафелю»
- «пришёл на смену обычным обоям»
- «вытесняет натуральный камень с рынка»
- «обогнал керамогранит по продажам»
- «эффективнее классических методов»
- «более технологичный чем традиционные материалы»
- «устаревшие материалы типа ДСП»
- «примитивные решения вроде вагонки»

**Как Haiku решает:** сравнения в пользу нашего материала — **ok**, если это **честное перечисление характеристик**. Не ok — **обесценивание конкурирующих материалов** («устаревший», «примитивный», «обычный»).

Ссылки на «успешный опыт», «тысячи клиентов», «популярность».

**Буквальные (layer 1 ловит числа):**
- «тысячи довольных клиентов»
- «95% наших заказчиков»

**Семантические перифразы:**
- «среди наших клиентов — известные отели»
- «устанавливали в элитных квартирах»
- «выбирают ведущие дизайнеры»

- «самый востребованный материал последних лет»

- «популярность растёт с каждым годом»
- «рекордный спрос»

**Как Haiku решает:** любая **социальная доказательность** без конкретики (кто именно, когда, какой объект) — нарушение. Разрешено: «на одном из проектов в Алуште — гостиница на 50 номеров...» (если это факт из KB с кейсом).

## Работа с False Positives

Haiku на границе категорий может срабатывать на легитимный контент. Примеры:

- «Материал экологичнее МДФ» — это **корректное сравнение**, не обесценивание. Категория 8 НЕ должна срабатывать.
- «Толщина 5 или 8 мм» — это **факт**, не маркетинг.
- «Срок службы типичен для композитных материалов» — это **общее заявление**, не обещание конкретного срока.

Если Haiku в вашем тестировании даёт много false positives — **ослабить** категории через дополнительные инструкции в промпте: «реагировать только на явные намёки на здоровье/обещания сроков/обесценивание конкурентов».

## Интеграция

```python
# Псевдокод для команды A
async def layer3_check(article_blocks: list[dict]) -> list[Violation]:
    all_text = [b["text"] for b in article_blocks if b.get("text")]
    corpus = all_text.join("\n\n")
    result = await claude_haiku.generate(
        system=LAYER3_SYSTEM_PROMPT,  # см. секцию "Промпт для Claude Haiku"
        messages=[{"role": "user", "content": corpus}],
        response_format={"type": "json"}
    )
    return parse_violations(result)
```

Вызывать **перед** `blog_publish` отправкой. Если есть violations — оператору A показать предупреждения (как сейчас с layer 1+2). Оператор решает: опубликовать, перегенерировать, отредактировать вручную.

## Эволюция корпуса

1. **Текущий список (10 категорий)** — стартовый, из моей практики
2. **После 10 production-статей** — добавить новые категории по реальным нарушениям которые проскочили
3. **Раз в месяц** — ревизия, удалить категории которые за весь период не сработали ни разу
